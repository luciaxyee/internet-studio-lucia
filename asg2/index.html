Structured Content verses Semantic HTML

Structured content is organization for humans to understand and semantic html is structure for the computer to understand. The information is more directly abvalible by machines, where in semantic html the search engine looks for the lower hierarchical information (ex. h3, h4, etc.) below the higher informational topic (in higher headers ex. h1, h2) it categorizes it. In structured content the machine looks more at the direct informaion and in semantic html it looks more at the coding structure.

Working together

They work together because structured content provides something for the seach engine to read, while the semantic allows the user to understand what the search engine/found wesite gives.

Importance

It's important to have structured data so that the computer knows what the site is about and is able to give the site as a suggestion when the user searches relating/correlating key words, and it's important to have semantic data so that when the user recieves the data they're able to understand the data given to them, and so that it's more structed in a way that makes the information more digestable.
